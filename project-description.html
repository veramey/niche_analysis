TenderGuru integration

Подробное описание проекта TenderGuru Integration

Этот репозиторий автоматизирует полный цикл работы с TenderGuru: поиск интересующих тендеров, сбор карточек закупок, скачивание прикрепленных документов, извлечение текста из файлов, анализ содержимого и экспорт результатов в удобные форматы. Если коротко, проект нужен для того, чтобы превратить поток закупок в структурированную базу для последующего анализа.

Node.js 18+ ESM Без внешних зависимостей CLI + API client

Как устроен поток работы

1

Подключение к API

Скрипты загружают переменные окружения из .env, после чего создается клиент TenderGuru с поддержкой api_code и refresh_code.

2

Поиск и фильтрация

Скрипт discovery перебирает набор поисковых запросов по AI, ML, RPA, computer vision, документам и автоматизации, затем отсеивает лишнее и оставляет релевантные тендеры.

3

Получение деталей

По каждой закупке собираются поля карточки: заказчик, цена, регион, сроки, ссылки на документы, номер тендера и данные контрагента.

4

Загрузка документов

Документы загружаются из ссылок, найденных в карточке тендера. Скрипт умеет проходить по вложенным HTML-страницам и раскрывать файлы глубже, если они спрятаны за промежуточными просмотрщиками.

5

Извлечение текста

Если скачан .docx, из него достается текст из word/document.xml, чтобы дальше работать не с бинарником, а с содержимым документа.

6

Анализ и экспорт

Текст документов анализируется по терминам, модулям и ограничениям, после чего формируются CSV, JSONL и Markdown-отчеты с выводом о продуктовой возможности.

Основные модули

src/tenderguru-client.js

Обертка над TenderGuru API v2.3. Она собирает параметры запросов, вызывает экспортные методы и возвращает ответы в том виде, в котором их дальше обрабатывают CLI и скрипты.

src/cli.js

Командная строка проекта. Она разбирает аргументы, запускает нужный API-вызов, редактирует секреты в выводе и печатает результат в консоль.

src/env.js

Загрузка переменных окружения из .env. Это позволяет хранить ключи отдельно от кода и не коммитить секреты в репозиторий.

src/export-utils.js

Набор общих утилит: создание директорий, очистка текста, редактирование секретов, запись CSV/JSONL и работа с датами.

Что делает каждый скрипт

scripts/discover-ai-automation-market.js

Ищет тендеры по набору AI/автоматизационных запросов, объединяет совпадения из поиска и раздела документации, присваивает баллы релевантности и пишет отобранные кандидаты в CSV/JSONL.

scripts/download-tender-docs.js

Берет отобранные тендеры, находит ссылки на документы, скачивает файлы, раскручивает вложенные HTML-страницы и извлекает текст из .docx.

scripts/analyze-tender-docs.js

Читает скачанный текст, ищет AI-термины, типы модулей, интеграции и ограничения, затем формирует продуктовые карточки и shortlist возможностей.

scripts/export-selected-tenders-docs.js

Экспортный пример: собирает документы по заранее заданным тендерам и сохраняет их в exports/ как CSV для дальнейшего просмотра.

Как выглядит полный сценарий работы

1

Поиск рынка

Запускается discover-ai-automation-market.js. Он перебирает набор ключевых фраз вроде искусственный интеллект, машинное обучение, RPA, workflow, распознавание документов и похожие формулировки. Так репозиторий не полагается на один запрос, а ищет рынок по многим вариантам написания.

2

Объединение результатов

Результаты поиска и результаты из раздела /documentation складываются в одну коллекцию кандидатов. Один и тот же тендер может встретиться в нескольких запросах, поэтому проект нормализует ID, номера карточек и ссылки, чтобы не дублировать записи.

3

Оценка релевантности

Каждому кандидату назначается score. В расчет идут совпадения по ключевым словам, наличие AI/ML/automation признаков, группа запроса и общая близость к теме автоматизации. После этого отбираются только релевантные тендеры.

4

Скачивание документов

Следующий этап - download-tender-docs.js. Скрипт читает CSV с кандидатами, собирает ссылки на файлы из карточек, скачивает их в data/raw/tenderguru/<date>/docs/ и пишет рядом метаданные о типе файла и исходном URL.

5

Рекурсивное раскрытие вложений

Если скачанный файл оказывается HTML-страницей просмотра, скрипт находит вложенные ссылки и скачивает их тоже. Это нужно, потому что на тендерных площадках документ нередко лежит за цепочкой промежуточных страниц и viewer-форм.

6

Извлечение текста

Когда файл является .docx, из него извлекается текст без внешних библиотек через unzip и XML внутри архива. Это превращает бинарный документ в обычный текстовый файл .txt, пригодный для анализа.

7

Анализ содержания

analyze-tender-docs.js берет текст документов и ищет признаки конкретного продукта: LLM/RAG-помощник, computer vision, медицинский AI, аналитическая ГИС, процессная автоматизация и другие паттерны. На этой основе формируется классификация и продуктовая интерпретация закупки.

8

Выходные артефакты

Результат пишется в CSV, JSONL и Markdown: это удобно и для машинной обработки, и для ручного чтения. По итогам можно быстро увидеть, какие закупки подходят под повторяемый AI-продукт, а какие требуют кастомной разработки.

Команды запуска

Проверка API

npm run smoke

Поиск тендеров

npm run tg -- search-tenders --kwords '"ремонт дороги"' --price1 10000000 --param f=44 --param r50=1

Карточка тендера

npm run tg -- tender --id 12402324

Контрагент

npm run tg -- contragent --inn 7716615618

Контакты контрагента

npm run tg -- contragent-contact --inn 7716615618

Произвольный запрос

npm run tg -- get --param mode=regions

Настройка окружения

Переменные

  • TENDERGURU_API_CODE - ключ доступа к API для платных полей и разделов.
  • TENDERGURU_REFRESH_CODE - код для получения или обновления api_code.

Файлы конфигурации

  • .env - локальные секреты и настройки.
  • package.json - команды npm и ограничение на Node.js 18+.

Где лежат результаты

data/raw/tenderguru/

Сырые выгрузки по датам: результаты поиска, карточки, HTML-просмотры, оригинальные документы и метаданные скачивания.

data/analysis/

Аналитический слой: CSV, JSONL, Markdown-отчеты, карточки продукта, списки ниш и MVP-гипотезы.

exports/

Готовые пользовательские выгрузки. Это точка, где лежат компактные таблицы для ручного просмотра или передачи дальше.

scripts/

Сценарии для поиска, скачивания, анализа и экспорта. Они связывают API-клиент с локальными папками и делают весь конвейер воспроизводимым.

Что получится на выходе

Структурированные таблицы

CSV-файлы с карточками тендеров, кандидатами, классификацией и итогами анализа. Их можно открыть в Excel, BI-системе или загрузить в другую обработку.

Текстовые отчеты

Markdown-файлы с короткими выводами: summary по нишам, shortlist возможностей, продуктовые карточки и пояснения по каждой закупке.

Сырые артефакты

HTML, DOCX, TXT и JSONL остаются рядом, чтобы можно было перепроверить исходные данные и понять, на чем основан вывод.

Повторяемый pipeline

Все шаги запускаются отдельными командами, поэтому можно повторить поиск, перекачать документы или заново пересчитать анализ за конкретную дату.
