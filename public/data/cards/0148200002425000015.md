## Бизнес-Боль

отчет «Автоматизация», содержащий информацию о количестве полностью автоматизированных обращений пользователей за выбранный период; строить сравнительные отчеты «Качество диалогов», «Количество диалогов», «Количество вопросов», «Популярность тем» и «Автоматизация» по двум сохраненным выборкам сессий; Автоматизация предусмотрена только для интентов (подтематик с семантически единым ответом), по которым в предоставленных Заказчиком вопросно-ответных парах есть не менее 200 примеров (автоматизация до 200 частотных интентов). Для прочих интентов в сценарии должна быть предусмотрена маршрутизация на страницу услуги на РПГУ. По согласованию с Заказчиком по итогам испытаний в ответ системы по низкочастотным подтематикам Исполнителем может быть добавлена справочная информация из отрывков НПА и вопросно-ответных пар, маркированная примечанием с предупреждением о возможных неточностях.

## Сценарии

- Сценарий предназначается для автоматизации обработки запросов пользователей, связанных с предоставлением государственных услуг, информирования и консультирования граждан в рамках компетенции указанных ведомств:
- При запросе данной услуги должен реализовываться следующий сценарий (каждый этап сценария может быть пропущен при наличии необходимой информации в запросе пользователя):
- Из личного кабинета пользователя на портале РПГУ, если пользователь авторизован (адрес передается в данных запроса);
- Запрос у пользователя в ручном режиме если пользователь не авторизован.
- Необходимо уточнить хотел бы пользователь заказать выписку по адресу регистрации, отобразив соответствующий адрес в диалоговом окне, подтвердить адрес или предложить указать иной адрес.
- В ходе проведения испытаний работоспособности пользовательских сценарий производится оценка по тестовой части интегральной выборки в соответствии с методикой оценки, приведенной в п. 4.2.5.2. настоящих Технических требований.
- По результатам проведения испытаний работоспособности пользовательских сценариев, в случае полного их соответствия критериям оценки, приведенным в п. 4.2.5.2 настоящих Технических требований, сторонами подписывается протокол испытаний.
- Методика оценки качества разработанных пользовательских сценариев
