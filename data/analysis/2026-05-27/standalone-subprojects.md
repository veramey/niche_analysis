# Самостоятельные подпроекты из тендерных данных

Дата анализа: 2026-05-27  
Источник: 800+ тендеров (AI/автоматизация 2026-05-25 + сметный рынок 2026-05-27)

---

## Группа A — Смежные с MVP по сметам

Эти модули естественно дополняют MVP генерации смет и могут быть разработаны поверх того же стека.

| Модуль | Тендеров | Сложность | Ключевые запросы |
|--------|:--------:|-----------|-----------------|
| Проверка сметы на соответствие нормативам | ~61 | Medium | "проверка сметной", "экспертиза сметной стоимости" |
| RAG по нормативной базе (ГЭСН/ФЕР/ГОСТ/СП) | ~59 | Medium | "ГЭСН", "ФЕРы", "ТЕРы", "ФГИС ЦС" |
| Генерация документа по шаблону (ЛСР, акт) | ~26 | Low | "сметная документация", "разработка ПСД" |
| OCR + извлечение полей (чертёж/ТЗ → объёмы) | ~46 | Medium | "проектно-сметная документация", "BIM смета" |

### Детали

**Проверка сметы на соответствие нормативам**  
Автоматическая сверка позиций сметы с актуальными расценками ГЭСН/ФЕР/ТЕР и требованиями ФГИС ЦС. Частая потребность у госзаказчиков и проверяющих органов. Встраивается в смета-MVP как отдельная вкладка «Проверить».

**RAG по нормативной базе**  
Чат-интерфейс поверх векторной базы из ГЭСН, ФЕР, ТЕР, ГОСТ, СП, приказов Минстроя. Вопрос → ссылка на пункт норматива + цитата. Особо востребован у сметчиков и экспертиз. Переиспользует инфраструктуру RAG из корпоративных LLM-ассистентов.

**Генерация документа по шаблону**  
Заполнение ЛСР, КС-2, КС-3, актов приёмки по структурированным данным сметы. Низкая сложность — шаблонный движок + нормализация данных. Высокая востребованность при работе с несколькими системами-источниками.

**OCR + извлечение полей**  
Загрузка PDF-чертежа или ТЗ → автоматическое извлечение объёмов работ, спецификаций, помещений. Подаётся на вход MVP для первичного заполнения сметы. Снижает ручной ввод на 60–80%.

## Группа B — Высокочастотные самостоятельные модули

Независимые продукты с большим числом тендеров, не требующие сметной экспертизы.

| Модуль | Тендеров | Сложность | Модель реализации |
|--------|:--------:|-----------|-------------------|
| Классификация + маршрутизация обращений | ~142 | Medium | SaaS / встраиваемый виджет |
| RPA-бот (перенос данных между системами) | ~77 | Low | Шаблонные сценарии |
| Document Summarizer (суммаризация + автозаполнение СЭД) | ~75 | Low | API-сервис |
| Chatbot для граждан (муниципалитеты) | ~71 | Low | **1 продукт × 85 регионов** |
| Мониторинг изменений на портале + алерт | ~38 | Low | Cron + webhook |
| Regulatory Document Analyzer (НПА → требования) | ~16 | Medium | RAG специализированный |

### Детали

**Классификация + маршрутизация обращений** (~142 тендера)  
Входящее обращение гражданина или сотрудника → автоматическая категория → нужное подразделение/очередь. Самый частый запрос в категории process_automation. Применимо в ЖКХ, МФЦ, call-центрах, службах поддержки.  
_Конкуренты_: PIX, UiPath, отечественные RPA-платформы — но кастомная классификация под домен выигрывает по качеству.

**RPA-бот** (~77 тендеров)  
Перенос и синхронизация данных между легаси-системами (1С ↔ ГИС ЖКХ, ЕИС, региональные порталы). Высокая повторяемость задачи, низкая техническая сложность. Продаётся как библиотека готовых сценариев + SLA на поддержку.

**Document Summarizer** (~75 тендеров)  
Загрузка договора/НПА/протокола → summary + автозаполнение карточки в СЭД. Прямая интеграция с Directum, ЭДО Диадок, 1С-Документооборот. Уже решаемая задача через GPT API — конкурентное преимущество в локализации и безопасности данных.

**Chatbot для граждан** (~71 тендер)  
Типовой FAQ-бот + эскалация на оператора для муниципальных портала и МФЦ. Стратегия: одна кодовая база, белый лейбл, региональный контент-пакет.  
_Оценка_: 85 регионов × 3–5 муниципалитетов = 250–400 потенциальных клиентов в РФ.

**Мониторинг изменений на портале** (~38 тендеров)  
Отслеживание изменений на ГИС-порталах, ЗакупкиСмарт, официальных сайтах → алерт в Telegram/Email. Простая реализация (cron + diff + webhook), высокая ценность для юристов, закупщиков, аудиторов.

**Regulatory Document Analyzer** (~16 тендеров)  
Загрузка НПА (приказ, ГОСТ, СП, постановление) → автоматическое извлечение требований, сроков, штрафов, ответственных. Более узкая ниша, но высокий ARPU — целевые клиенты: юридические службы, строительные экспертизы, органы надзора.

## Наблюдения

### Паттерн "1 продукт × N регионов"
Chatbot для граждан и Классификация обращений реализуются один раз и тиражируются горизонтально. Прецеденты в данных: Webiomed (медицина, 20+ регионов), боты для ЖКХ (Башкортостан, Татарстан, Московская обл.). Регионы бюджетируют независимо — один и тот же продукт выигрывает тендеры в 5–10 субъектах РФ.

### Сметный рынок — отдельная вертикаль
Прямых тендеров на лицензии сметного ПО найдено ~34 (бюджеты 100к–16.8М руб). Это _не_ рынок для нового ПО — Гранд-Смета и Адепт:Смета уже укоренились. Точка входа — AI-надстройки (OCR, проверка, RAG по нормативам), а не замена существующих инструментов.

### Медицинский AI — крупнейший сегмент, высокий барьер
55+ тендеров в `medical_ai_support`, бюджеты до 609М руб. Требуются: регистрационное удостоверение Росздравнадзора, КТ/МРТ датасеты, партнёрство с медучреждением. Не рекомендуется как первый продукт.

### Безопасные RAG/LLM — повторяющийся паттерн
15+ тендеров на корпоративные LLM-ассистенты (100М–600М руб). Требование: развёртывание on-premise или в ГОСОблаке, совместимость с YandexGPT/GigaChat, аттестация ФСБ для секретных данных. Рынок реальный, но регуляторный барьер высокий.

## Рекомендация по приоритету

```
Уровень 1 (быстрый старт, смежный с MVP):
  → RAG по нормативной базе (ГЭСН/ФЕР/СП)
  → Проверка сметы на соответствие нормативам

Уровень 2 (независимый продукт, тиражируемый):
  → Chatbot для граждан (муниципалитеты)
  → Классификация + маршрутизация обращений

Уровень 3 (более широкий рынок, выше конкуренция):
  → Document Summarizer
  → RPA-бот

Отложить:
  → Медицинский AI (регуляторный барьер)
  → Корпоративный RAG/LLM on-premise (длинный цикл сделки)
```
