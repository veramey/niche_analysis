# Самостоятельные подпроекты внутри AI/automation тендеров

Основа анализа:

- `data/analysis/2026-05-25/tender-requirements.jsonl`
- `data/analysis/2026-05-25/product-cards/`
- корпус: 599 структурированных тендеров и 600 карточек

Под "самостоятельным подпроектом" здесь понимается модуль, который можно продать отдельно от большого тендера: у него есть свой пользователь, входные данные, результат, метрика ценности и ограниченный MVP.

## Короткий вывод

В крупных тендерах часто спрятаны не один, а несколько продуктов:

- AI-разбор документов;
- OCR и извлечение данных;
- ассистент для обращений;
- RPA-роботы для мониторинга порталов;
- контроль качества документов и ответов;
- разметка датасетов;
- эксплуатация и поддержка AI-сервисов;
- контент-анализ и дашборды;
- медицинский AI operations;
- CV-инспекция изображений и видео.

Для стартапа самые реалистичные самостоятельные подпроекты:

1. `AI document checker` - разбор требований, рисков и ошибок в документах.
2. `OCR + extraction layer` - извлечение полей из документов и передача в ЭДО/1С/API.
3. `Appeals assistant` - классификация обращений и черновики ответов.
4. `RPA monitor bot` - робот для мониторинга порталов и реестров.
5. `Dataset preparation service` - разметка и подготовка данных для ML.

## 1. AI-разбор требований и рисков в документах

Что можно вынести:

- загрузка ТЗ, НПА, ЛНА, договора, конкурсной документации;
- извлечение обязательных требований;
- поиск рисков, сроков, штрафов, противоречий;
- чеклист исполнения;
- вопросы к документу на естественном языке.

Почему это самостоятельный продукт:

- не требует тяжелой интеграции на первом этапе;
- результат можно показать сразу после загрузки документа;
- подходит юристам, закупщикам, подрядчикам и compliance.

Примеры тендеров:

- `91793258` - разработка ПО для анализа требований нормативных документов с ИИ и языковыми моделями.
- `91794551` - аналогичный запрос по анализу нормативных документов.
- `0373100103725000018` - аналитическая AI-система с чат-ботом по НПА/ЛНА и шаблонами документов.
- `0173100011226000001` - Росфинмониторинг, большой контур с LLM и обработкой входящих документов.

MVP:

- загрузка PDF/DOCX;
- извлечение требований;
- таблица рисков;
- summary;
- экспорт в DOCX/XLSX.

## 2. OCR и извлечение данных из документов

Что можно вынести:

- распознавание сканов;
- извлечение реквизитов, сумм, дат, контрагентов;
- валидация полей;
- выгрузка в ЭДО, 1С, Диадок, Excel или API.

Почему это самостоятельный продукт:

- у заказчиков уже есть документы и понятная боль;
- можно стартовать с одного типа документа;
- легко измерять эффект: скорость обработки, доля ошибок, ручные часы.

Примеры тендеров:

- `90680793` - OCR-система для Азиатско-Тихоокеанского Банка.
- `88409850` - внедрение платформы ЭДО с OCR, интеграцией с 1С и Диадок.
- `88519611` - аналогичная платформа ЭДО с модулем распознавания документов.
- `0190200000325015669` - доступ к ПО для автоматизированного распознавания документов.
- `0830500000225001923` - поставка ПО для автоматизации распознавания документов.

MVP:

- загрузка документа;
- OCR;
- извлечение 10-20 ключевых полей;
- ручная проверка;
- экспорт.

## 3. AI-ассистент для обработки обращений

Что можно вынести:

- классификация обращений;
- маршрутизация;
- черновик ответа;
- summary истории обращения;
- контроль SLA;
- эскалация сложных случаев оператору.

Почему это самостоятельный продукт:

- не обязательно автоматизировать весь контакт-центр;
- можно начать с подсказок оператору;
- понятна ценность для ЖКХ, медицины, МФЦ, ведомств и сервисных компаний.

Примеры тендеров:

- `32616004003` - Мособлгаз, система формирования ответов на обращения на базе ИИ.
- `32616037287` - Уфаводоканал, "Робот Диспетчер ЖКХ" для обработки обращений жителей.
- `32615847668` - омниканальный контакт-центр с голосовыми и неголосовыми обращениями.
- `0830500000225004235` - голосовой помощник для записи к врачу.
- `0187300015625000041` - нейронная сеть помощи гражданам и официальным ресурсам.

MVP:

- входящие письма/заявки;
- классификация темы;
- рекомендованный ответ;
- confidence score;
- кнопка "отправить/доработать";
- отчет по темам и SLA.

## 4. RPA-робот для мониторинга порталов и рутины

Что можно вынести:

- робот, который ходит на внешний портал;
- собирает данные;
- проверяет изменения;
- формирует отчет;
- сигналит об ошибках;
- обновляет сценарий при изменении сайта.

Почему это самостоятельный продукт:

- часто не нужна интеграция с внутренними системами;
- можно автоматизировать один повторяемый процесс;
- заказчик покупает экономию ручного труда.

Примеры тендеров:

- `0137200001225007767` - мониторинг раскрытия информации РСО на портале `ri.eias.ru` с RPA.
- `0137200001225005799` - автоматизированный процесс на базе OCR и программируемых роботов.
- `85696429` - сопровождение и модификация программных роботов Robin RPA.
- `32515512927` - поддержка программных роботов Sherpa RPA.
- `90363698` - лицензии на PIX RPA.

MVP:

- настройка одного робота;
- расписание запусков;
- журнал ошибок;
- отчет в Excel/почту;
- простой UI для статуса.

## 5. Контроль качества документов и AI-ответов

Что можно вынести:

- проверка орфографии, пунктуации и стиля;
- проверка комплектности проекта документа;
- обнаружение несоответствий правилам;
- валидация AI-сгенерированных ответов перед отправкой;
- журнал правок и протоколирование.

Почему это самостоятельный продукт:

- это уже не "генератор текста", а контроль перед выпуском;
- хорошо ложится в ЭДО и юридический документооборот;
- проще продавать как снижение ошибок и рисков.

Примеры тендеров:

- `0146500000325000012` - Липецкая область, проверка проектов документов в ЭДО на орфографию и пунктуацию через нейросеть.
- `0373100103725000018` - чат-бот по НПА/ЛНА, генерация шаблонов документов и проверка текстов.
- `32616004003` - генерация ответов на обращения с классификацией случаев, требующих ручной обработки.

MVP:

- загрузка/получение проекта документа;
- правила проверки;
- AI-комментарии;
- список исправлений;
- протокол проверки.

## 6. Разметка датасетов и подготовка данных

- заказчикам нужны данные до разработки модели;
- можно продавать как услугу и постепенно продуктовать;
- не нужно отвечать за весь production AI-контур.

- кабинет проверки;
- контроль согласованности меток;

## 9. Computer Vision инспекция

Что можно вынести:

- распознавание номеров;
- анализ изображений и видео;
- кабинет проверки результатов;
- отчет качества;
- API для передачи событий.

Почему это самостоятельный продукт:

- в тендерах много CV-задач;
- но лучше заходить вертикально: дороги, транспорт, медицина, безопасность, производство;
- без вертикали легко стать подрядчиком по кастомной интеграции.

Примеры тендеров:

- `0101500000325000150` - фотовидеофиксация и распознавание регистрационных знаков.
- `0101500000325000345` - содержание системы фотовидеофиксации.
- `0101500000326000125` - аналогичное содержание системы фотовидеофиксации.
- `0356500001425006618` - компонент компьютерного зрения для мониторинга дорожной сети.
- `0356500001425006620` - аналогичный CV-компонент.

MVP:

- прием фото/видео;
- модель распознавания;
- кабинет ручной проверки;
- журнал событий;
- API/webhook.

## Приоритет для стартапа

### Самый простой вход

1. AI-разбор требований и документов.
2. OCR + extraction layer.
3. AI-ассистент для обращений.
4. RPA monitor bot.
5. Контроль качества документов.

### Самый высокий чек

1. RAG/LLM база знаний.
2. AI operations для медицинских и гос AI-сервисов.
3. Контент-анализ и управление задачами.
4. CV-инспекция.

### Лучшее сочетание простоты и потенциала

**AI document checker для закупок, регламентов и договоров.**

Первый продукт:

- загрузка ТЗ/договора/НПА;
- extraction требований;
- risk checklist;
- вопросы к документу;
- экспорт в таблицу.

Первый платящий сегмент:

- компании, которые регулярно участвуют в тендерах;
- юристы и тендерные отделы;
- интеграторы, которые разбирают ТЗ перед подачей заявки;
- подрядчики в госзакупках.
